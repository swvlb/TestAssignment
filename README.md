# TestAssignment

Тестовое для ШИФТ Лаб. SwiftUI + MVVM, весь интерфейс в красной фирменной палитре.

Слои разложены по папкам: Domain (модели и протоколы), Data (сеть и хранилище), а фичи в RegistrationModule и MainModule. Сеть спрятана за протоколом ProductsRepository и наружу отдаёт доменную модель, маппинг из ProductDTO живёт в Data. За счёт этого вью-модели ничего не знают про JSON и спокойно тестируются с моком.

## Экраны

Регистрация: имя, фамилия, дата рождения, пароль и повтор. Дату выбираю через UIDatePicker, обёрнутый в UIViewRepresentable, заодно показал, что связка UIKit и SwiftUI не проблема. Проверяю поля на каждый ввод, но ошибку под полем подсвечиваю только после того, как из него ушли, чтобы не краснить пустую форму сразу.

Главный экран: список товаров с fakestoreapi.com (название и цена), грузится на async/await, есть состояния загрузки и ошибки с кнопкой «Повторить». Кнопка «Приветствие» открывает модалку с приветом по имени, «Выйти» просит подтверждение через алерт и сбрасывает сессию.

## Сессия

Один раз зарегистрировался и дальше сразу попадаешь на главный экран. Флаг и имя лежат в UserDefaults . Выход прикрутил, чтобы не сносить приложение каждый раз ради повторного прохода, при проверке так удобнее.

## Валидация

Правила собрал в RegistrationValidators, каждое поле проверяется своей функцией и возвращает текст ошибки либо nil:

- имя и фамилия от двух символов, только буквы и дефис
- дата рождения не из будущего, возраст от 14 до 120
- пароль от восьми символов, хотя бы одна цифра и одна заглавная
- повтор совпадает с паролем

Пример проходящего пароля: 
```
Qwerty123!
```

## Тесты

XCTest на всю логику: валидаторы, обе вью-модели (в том числе загрузка товаров и выход) и хранилище сессии через мок. Запуск из Xcode по Cmd+U либо make test.

## Запуск

Проект генерится XcodeGen, .xcodeproj в репозиторий не кладу. Ставится один раз:

    brew install xcodegen


Остальное через Makefile:

    make open     // сгенерировать и открыть в Xcode
    make build    // собрать из терминала
    make test     // прогнать тесты
